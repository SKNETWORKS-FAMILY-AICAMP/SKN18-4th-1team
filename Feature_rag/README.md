
# vector 적재방법
- docker-compose.yml 실행
- ./service/ingest_doc.py 파일 실행

사용자 질문 (자연어)
      ↓
[1] LLM -> 분류(과분류/할 수 있는 질문인지 판단)
      ↓
[2] RAG -> 증상에 대한 구체화(의심 질환 후보 조회)
      ↓
[3] LLM → 상황 판단 
    - 가장 가능성 높은 질환 선정
    - 중증도 판단
    - 최종 진료과 확정
      ↓
[3] RDB → 병원 필터링
    (1) 진료과 일치
    (2) 위치 제한 (사용자 시/구 기준)
    (3) 검사/치료에 필요한 장비 보유 여부
    (4) 병원 종별 조건 (상급 종합 필요 여부)
      ↓
[4] 점수모델 → 병원 순위화
병원 점수 = 진료과 일치 점수
          + 장비 가중치
          + 간호등급 점수
          + 병원 규모 점수
          - 거리 패널티
      ↓
[5] LLM → "왜 이 병원이 적합한지" 설명 생성
